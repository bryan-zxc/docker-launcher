# ADR-0004: JSON file for container metadata

## Context

Container lifecycle data (image, repo URL, creation time) lives in Docker labels, but some metadata — like `last_opened_at` — is generated by the app and has no natural home in Docker. A lightweight persistence layer is needed for this app-side metadata.

## Alternatives Considered

- **SQLite** — provides querying, transactions, and schema enforcement, but adds a binary dependency and is heavier than needed. The metadata is simple key-value data with no relational queries.
- **Docker labels** — cannot be updated after container creation without recreating the container. Not suitable for mutable metadata.

## Decision

Use a single JSON file (`~/.docker-launcher/containers.json`) as the metadata store. Each key is a container ID mapping to a flat dict of metadata fields. The file is read and written in full on each operation.

Orphaned entries (containers deleted outside the app) are cleaned up automatically when the container list is fetched.

## Consequences

- No additional dependencies — JSON is in the standard library.
- The file lives in `~/.docker-launcher/`, shared with other app data (settings).
- Not suitable for high-concurrency writes, but acceptable for a single-user local tool.
